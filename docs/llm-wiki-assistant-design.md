# LLM Wiki Assistant 设计文档

## 1. 项目概述

基于 [llm-wiki.md](file:///workspace/docs/llm-wiki.md) 的理念，设计一个个人使用的知识库管理AI工具，使用 Agno 框架实现。该工具将帮助用户构建和维护一个由 LLM 驱动的持久化知识库，实现知识的增量积累和智能管理。

## 2. 核心设计理念

### 2.1 三层架构

- **原始源文件 (Raw sources)**：用户的原始文档、文章、笔记等，作为知识库的数据源
- **Wiki 层**：由 LLM 生成和维护的结构化 markdown 文件集合，包含摘要、实体页面、概念页面等
- **Schema 层**：定义 wiki 结构、约定和工作流程的配置文件

### 2.2 核心功能

- **知识摄入**：处理新的源文件，提取关键信息并集成到现有 wiki 中
- **知识查询**：基于 wiki 内容回答用户问题，支持多种输出格式
- **知识维护**：定期检查 wiki 健康状况，识别矛盾、过时信息等
- **索引与日志**：维护 index.md 和 log.md 文件，方便导航和追踪

## 3. 技术选型

### 3.1 框架与依赖

| 类别 | 技术/库 | 版本 | 用途 | 选型理由 |
|------|---------|------|------|----------|
| 核心框架 | Agno | 最新版 | 构建AI代理系统 | 提供完整的知识管理、多代理和工具集成能力 |

| 存储格式 | Markdown | - | Wiki内容存储 | 简洁易读，支持丰富的格式 |
| 搜索引擎 | qmd | 最新版 | Wiki内容搜索 | 本地搜索，支持混合BM25/向量搜索 |

## 4. 系统架构

### 4.1 模块设计

1. **知识管理模块**
   - 负责原始源文件的管理
   - 与向量数据库交互，处理嵌入和检索
   - 提供知识摄入和查询接口

2. **Wiki 管理模块**
   - 负责生成和维护 Wiki 页面
   - 处理页面更新、交叉引用等
   - 维护 index.md 和 log.md

3. **代理系统**
   - 基于 Agno Agent 构建
   - 集成知识管理和 Wiki 管理功能
   - 处理用户查询和指令

4. **工具集成**
   - 搜索工具：qmd 集成
   - 网络搜索：DuckDuckGo 集成
   - 其他实用工具

### 4.2 数据流

1. **知识摄入流程**
   - 用户添加新源文件到 raw/sources/
   - 系统读取文件内容
   - 生成嵌入并存储到向量数据库
   - LLM 分析内容，生成或更新 Wiki 页面
   - 更新 index.md 和 log.md

2. **知识查询流程**
   - 用户提交查询
   - 系统搜索相关 Wiki 页面
   - LLM 基于 Wiki 内容生成回答
   - 可选：将回答保存为新的 Wiki 页面

3. **知识维护流程**
   - 定期检查 Wiki 健康状况
   - 识别矛盾、过时信息、孤立页面等
   - 生成维护建议
   - 自动或手动执行维护操作

## 5. 功能详细设计

### 5.1 知识摄入功能

- **支持的源文件类型**：
  - Markdown 文件
  - 网页链接（通过网页剪取工具）
  - 纯文本文件
  - 图片（通过OCR处理）

- **摄入流程**：
  1. 读取源文件内容
  2. 提取关键信息和实体
  3. 生成摘要
  4. 更新相关 Wiki 页面
  5. 添加交叉引用
  6. 更新索引和日志

### 5.2 知识查询功能

- **查询类型**：
  - 事实性问题
  - 综合性问题
  - 比较性问题
  - 探索性问题

- **输出格式**：
  - Markdown 文本
  - 表格
  - 幻灯片（Marp）
  - 图表

### 5.3 知识维护功能

- **健康检查**：
  - 页面间矛盾检测
  - 过时信息识别
  - 孤立页面检测
  - 缺失交叉引用识别

- **维护操作**：
  - 自动修复简单问题
  - 生成维护建议
  - 提供手动维护接口

### 5.4 索引与日志

- **index.md**：
  - 按类别组织 Wiki 页面
  - 包含页面链接和简短描述
  - 支持元数据（日期、源文件数等）

- **log.md**：
  - 按时间顺序记录操作
  - 包含摄入、查询、维护等事件
  - 支持前缀搜索和过滤

## 6. 技术实现细节

### 6.1 Agno 集成

- **知识管理**：使用 Agno 的 Knowledge 类
- **向量数据库**：配置 LanceDB 作为存储
- **代理系统**：构建专用的 Wiki 管理代理
- **工具集成**：集成搜索和网络工具

### 6.2 配置与环境

- **环境变量**：
  - OPENAI_API_KEY：OpenAI API 密钥
  - LANCE_DB_PATH：LanceDB 存储路径
  - WIKI_PATH：Wiki 存储路径
  - RAW_PATH：原始源文件路径

- **配置文件**：
  - config/schema.md：定义 Wiki 结构和工作流程
  - .env：环境变量配置

### 6.3 性能优化

- **嵌入缓存**：避免重复生成嵌入
- **增量更新**：只处理变更的内容
- **并行处理**：利用 Agno 的多代理能力
- **搜索优化**：配置合适的搜索参数

## 7. 部署与使用

### 7.1 安装与配置

1. 安装依赖：`pip install -r requirements.txt`
2. 配置环境变量
3. 初始化知识库：`python src/init.py`

### 7.2 使用流程

1. **添加源文件**：将文件放入 raw/sources/ 目录
2. **摄入知识**：运行 `python src/ingest.py` 或通过 API 调用
3. **查询知识**：运行 `python src/query.py "你的问题"` 或通过 API 调用
4. **维护知识**：定期运行 `python src/maintain.py`

### 7.3 集成与扩展

- **Obsidian 集成**：作为 Wiki 查看和编辑工具
- **Web 界面**：提供简单的 Web 界面
- **API 接口**：支持外部应用集成
